# PA Agent 服务端部署指南（NAS / Docker）

把 PA Agent 部署到 NAS（或任意 Linux 服务器）上后台运行：多品种轮巡分析 +
飞书推送 + 浏览器管理页。桌面 GUI 版不受影响，可继续独立使用。

## 功能一览

- **多品种并发轮巡**：按配置的品种列表并发执行两阶段 AI 分析（同时分析数
  可配 1-8，默认 2），一轮结束后按设定间隔自动开始下一轮；单品种失败自动
  跳过，不会卡死。
- **仅交易时段轮巡**（可选）：休市（周末/时段外）自动挂起、开盘自动恢复，
  节省 API 额度；时段可自定义（默认 A股+港股并集）。
- **飞书推送**：分析命中「下单机会」（限价单/突破单/市价单，且信心达到阈值）
  时推送飞书卡片，配置了 app_id/app_secret 时附带 K 线图。
- **Web 管理页**：`http://NAS_IP:8688`——监控（并发状态/结果/事件日志）、
  品种详情（K 线图 + 决策价位标注 + 实时推理过程）、配置（模型/飞书/监控
  列表/数据源）、历史（历次分析记录、K 线图与详情）。

## 前置条件

- NAS 支持 Docker（群晖套件中心装 Container Manager；威联通装 Container
  Station；绿联/极空间等在应用中心找 Docker）。
- NAS 能访问外网（拉取行情与调用 AI 接口）。
- 数据源用 TradingView / 东方财富 / AkShare / Tushare。**MT5 不支持**
  （仅 Windows 桌面版可用）。

## 部署步骤

1. 把仓库上传到 NAS（或在 NAS 上 `git clone`），进入仓库目录。
2. 构建并启动：

   ```bash
   docker compose -f docker-compose.server.yml up -d --build
   ```

   首次构建需拉取依赖，约 5-15 分钟。
3. 浏览器打开 `http://NAS_IP:8688`：
   - 「配置」页填 AI 模型（base_url / model / API Key）；
   - 填飞书 Webhook（飞书群 → 设置 → 群机器人 → 添加「自定义机器人」），
     点「发送测试消息」确认连通；
   - 填监控品种列表（如 `XAUUSD, BTCUSD, EURUSD`）、K 线周期、轮巡间隔；
   - 保存配置。
4. 回「监控」页，点右上角「启动轮巡」。

## 数据与备份

所有运行数据都在仓库目录下的 `./data/`（首次启动自动创建）：

| 目录 | 内容 |
|---|---|
| `data/config/` | `settings.json`（含 API Key，注意保管） |
| `data/records/` | 历次分析完整记录 |
| `data/trade_records/` | 下单信号 CSV 与图表 PNG |
| `data/experience/` | 经验库（可手动放入经验文件） |
| `data/logs/` | 运行日志 |

备份 `data/` 目录即可；服务端配置与桌面 GUI 的 `config/settings.json`
相互独立，互不影响。

注意：`data/records/pending/` 会随轮巡持续累积分析记录（每次分析一个
JSON，含完整推理过程，单个数百 KB）。长期运行建议定期归档或删除过旧的
记录文件，删除不影响程序运行。

## 升级

```bash
git pull
docker compose -f docker-compose.server.yml up -d --build
```

`data/` 目录不受升级影响。

## 常见问题

- **端口 8688 被占用**：改 `docker-compose.server.yml` 的映射，如
  `"9000:8688"`，然后访问 `http://NAS_IP:9000`。
- **TradingView 连不上 / 连接超时 / 数据一直未就绪**：国内网络环境需要给
  容器配代理。在 compose 的 `environment` 中加（然后 `up -d` 重建容器）：

  ```yaml
  - HTTPS_PROXY=http://192.168.1.100:7890
  - HTTP_PROXY=http://192.168.1.100:7890
  - NO_PROXY=localhost,127.0.0.1,api.deepseek.com,.eastmoney.com,.feishu.cn,open.feishu.cn
  ```

  要点：① 地址填代理所在机器的**局域网 IP**（容器内 127.0.0.1 是容器自己，
  不要用）；② 代理客户端（Clash 等）要开「允许局域网连接」；③ 用 `http://`
  形式端口（Clash 7890 混合端口即可），别写 `socks5://`；④ `NO_PROXY` 让
  DeepSeek/东财/飞书等国内直连的流量不走代理。验证：
  `docker exec pa-agent-server python -c "import requests;
  print(requests.get('https://www.tradingview.com', timeout=10).status_code)"`
  输出 200 即通。
- **飞书测试失败**：确认 webhook 地址完整；机器人若开了「签名校验」，需在
  配置页填 secret；开了 IP 白名单则需把 NAS 出口 IP 加入。
- **轮巡启动报「未配置 API Key」**：在配置页 AI 模型区填 API Key 并保存。
- **日志出现「未检测到 WorkBuddy 环境」**：说明模型名填了 `openclaw_wb`
  前缀（WorkBuddy 桌面路由），服务器上没有该环境。用自己的 API Key 时把模型
  名填成普通模型（如 `deepseek-chat`）即可；若是分析中网络波动触发的降级
  尝试提示，可无视。
- **服务端不建议使用 openclaw 系网关模型**：其网络错误自动回退机制会改写
  共享配置，在多品种并发分析下不安全。服务端请使用直连 API 模型
  （DeepSeek / OpenAI 兼容接口 + 自己的 API Key）。
- **想在手机上用**：管理页是响应式的，手机浏览器直接访问同一地址即可
  （建议仅内网使用，勿把端口暴露到公网——管理页无登录鉴权）。
- **不用 Docker 直接跑**：`pip install -e ".[server]" && python -m
  pa_agent.server`（Python ≥ 3.11），数据将写在仓库目录本身的
  `config/`、`records/` 等目录下。
